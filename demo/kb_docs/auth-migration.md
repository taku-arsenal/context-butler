# 認証基盤移行プロジェクト

## 概要

社内ポータルの認証方式を独自認証から AWS Cognito 連携に移行するプロジェクト。

## 背景

- 現在の独自認証は運用負荷が高い（パスワードリセット対応・セッション管理など）
- セキュリティ監査で改善が求められている
- AWS 側に認証基盤を寄せることでコスト削減・セキュリティ向上が見込める
- Issue #123 で詳細な提案が記載されている

## 移行方針

1. まず開発環境で Cognito 連携を試験導入
2. 動作確認後、ステージング環境に適用
3. 本番環境への適用は別途判断（ステークホルダー合意が必要）

## 影響範囲

- ログイン処理（`src/auth/login.py`）
- ユーザー管理（`src/users/`）
- 権限チェック（`src/middleware/auth.py`）
- セッション管理（`src/auth/session.py`）

## 関連 Issue

- GitHub Issue #123: 認証方式を Cognito に移行する

## 決定事項

- 2026-05-03 定例にて、認証基盤を AWS 側に寄せる方針が合意
- まず開発環境で試験導入することが決定
- 本番切り替えは別途判断
