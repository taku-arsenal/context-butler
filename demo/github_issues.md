## Issue #123: 認証方式を Cognito に移行する

**ステータス**: In Progress  
**担当**: 開発チーム  
**作成日**: 2026-04-15  
**更新日**: 2026-05-03

### 概要

セキュリティ運用負荷を下げるため、認証基盤を AWS Cognito に移行する。

### 背景

- 現在の独自認証は運用確認に時間がかかる
- パスワードリセット対応・セッション管理の運用負荷が高い
- セキュリティ監査で改善が求められている
- AWS 側に認証基盤を寄せることでコスト削減・セキュリティ向上が見込める

### 影響範囲

- ログイン処理（`src/auth/login.py`）
- ユーザー管理（`src/users/`）
- 権限チェック（`src/middleware/auth.py`）
- セッション管理（`src/auth/session.py`）

### 対応方針

1. 開発環境で Cognito 連携を試験導入
2. 動作確認後、ステージング環境に適用
3. 本番適用は別途判断

### コメント

**2026-05-03 定例にて合意**:
- 認証基盤を AWS 側に寄せる方針が合意
- まず開発環境で試験導入することが決定
- 来週から開発環境での切り替えを開始予定
