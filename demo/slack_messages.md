# デモ用 Slack 投稿例

## パターン 1: 認証切り替えの共有（メイン）

### 悪い投稿（デモ用）

```
例の認証の件、来週から切り替える方向で。影響あるところだけ確認お願いします。
```

### 期待される補足文

```
補足です。この投稿は、社内ポータルの認証方式を既存の独自認証から Cognito（AWS の認証サービス）連携に切り替える件についての共有です。

**背景**
先週の定例（2026-05-03）でセキュリティ運用負荷を下げるため、認証基盤を AWS 側に寄せる方針が合意されています。Issue #123 でも認証遅延と確認漏れが課題として挙がっています。

**対象範囲**
今回の変更対象はまず開発環境です。本番切り替えは別途判断予定です。利用者向け通知ではなく、ログイン処理・ユーザー管理・権限チェックに関わる内部実装が対象です。

**確認すべき人**
ログイン処理・ユーザー管理・権限チェックに関わる実装を持つ担当者です。

**次に取るべき行動**
影響範囲を確認し、懸念点があれば来週の切り替え前にこのスレッドへ返信してください。問題なければ Issue #123 に影響範囲を追記してください。

**不確かな点**
本番適用の具体的な日程とロールバック手順は、この投稿とスレッドだけでは明記されていません。
```

## パターン 2: 仕様変更の通知

### 悪い投稿（デモ用）

```
通知の件、変更します。詳細は後で共有します。
```

### 期待される補足文

```
補足です。この投稿は、通知機能の仕様変更についての事前共有です。

**背景**
現在の通知機能に関して変更が予定されています。詳細は後日共有される予定です。

**不確かな点**
- 何の通知機能が対象か（メール通知？Slack 通知？プッシュ通知？）
- どのような変更が行われるか
- いつ詳細が共有されるか
- 誰が影響を受けるか

この投稿だけでは変更の詳細が不明です。詳細共有をお待ちください。
```

## パターン 3: 障害報告

### 悪い投稿（デモ用）

```
さっきのエラー、原因分かりました。対応します。
```

### 期待される補足文

```
補足です。この投稿は、先ほど発生したエラーの原因特定と対応開始についての報告です。

**不確かな点**
- どのエラーが対象か（エラーコード・発生時刻・影響範囲が不明）
- 原因が何か
- どのような対応を行うか
- 対応完了の見込み時刻が不明

この投稿だけでは詳細が不明です。エラーの詳細・原因・対応内容・完了見込みをスレッドに追記いただけると助かります。
```

---

## パターン 4: 依頼

### 悪い投稿（デモ用）

```
例のやつ、今週中にお願いします。
```

### 期待される補足文

```
補足です。この投稿は、何らかのタスクを今週中に完了するよう依頼するものです。

**不確かな点**
- 「例のやつ」が何を指すか（タスク名・内容が不明）
- 誰への依頼か
- 今週中とは具体的にいつまでか（金曜 EOD？）
- 完了後のアクション（報告先・方法）が不明

この投稿だけでは依頼内容が特定できません。タスクの詳細をスレッドに追記いただけると助かります。
```
