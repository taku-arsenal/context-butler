## 3. 制約条件

| 制約 | 内容 |
|------|------|
| チャネル | Slack のみ（LINE・Teams は将来対応） |
| 起動方法 | Message Shortcut を本命とする（メンション起動は開発初期の簡易版として許容） |
| Agent 構成 | AgentCore Runtime + Strands Orchestrator で 4 Unit / 4 Agent を順次制御することを MVP の第一候補にする |
| MCP | Google Drive と GitHub のみ（Slack MCP は使わない） |
| Slack 文脈取得 | Slack API を直接使う（Slack MCP は使わない） |
| A2A | MVP では使わない（将来拡張として設計） |
| Slack 権限境界 | MVP デモは権限のある参加者のみを招待した Private チャンネルで実施する |
| リージョン | ap-northeast-1（東京）を基本とする |

## 4. ユースケース

### UC-01: 説明不足な投稿を補足する（主要ユースケース）

```
アクター: Slack ユーザー（受け手）
前提条件: Slack チャンネルに説明不足な投稿がある
主フロー:
  1. ユーザーが分かりにくい投稿の三点リーダを押す
  2. 「Explain with 説明補足AI」を選択する
  3. （任意）モーダルで補足レベル・想定読者を選択する
  4. アプリが 3 秒以内に受付確認を返す
  5. バックグラウンドで 4 Unit の AI 処理が実行される
     5.1 Unit 1: 省略抽出 Agent が省略点を抽出する
     5.2 Unit 2: 文脈取得 Agent が必要文脈を収集する
     5.3 Unit 3: 補足文生成 Agent が補足文を生成する
     5.4 Unit 4: リテラシーレビュー Agent が品質を確認する
  6. 元投稿のスレッドに補足文が返信される
後条件: 受け手が追加質問せずに投稿の意図を理解できる
```

### UC-02: フィードバックを送る（将来対応）

```
アクター: Slack ユーザー
前提条件: 補足文が生成されている
主フロー:
  1. ユーザーが補足文のフィードバックボタンを押す
  2. 「役に立った」「長すぎる」「違う」などを選択する
  3. フィードバックが DynamoDB に記録される
後条件: フィードバックが品質改善に活用される
```

## 5. 画面・UX 要件

### 5.1 Message Shortcut

- Slack の三点リーダメニューに「Explain with 説明補足AI」が表示される
- 選択後、3 秒以内に何らかの応答がある（ack または「生成中です」メッセージ）

### 5.2 Slack モーダル（Should）

モーダルで以下を選択できる：

| 項目 | 選択肢 |
|------|--------|
| 補足レベル | かんたん / 標準 / 詳細 |
| 想定読者 | 新人 / 非エンジニア / エンジニア / プロジェクトメンバー |
| 例示を含めるか | あり / なし |
| 投稿先 | スレッドに返信 / 自分だけにプレビュー |

### 5.3 補足文の返信

- 元投稿のスレッドに返信される
- 補足文の冒頭に「補足です。」と明記する
- 不確かな点は「なお、〜はこの投稿だけでは明記されていません。」と明記する
- Slack のマークダウンを適切に使用する
